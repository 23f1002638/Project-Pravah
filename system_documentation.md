# Project Pravah: Technical System Documentation

This document serves as the developer-facing architecture manual for Project Pravah, detailing the machine learning pipelines, backend Flask web service, frontend Next.js interface, and data flow schemas.

---

## 1. Data Schema & Payload Structures

All data flowing from the client dashboard to the analytical backend must match the following structures.

### 1.1 Ingestion Schema (Client Request)
The backend endpoint `/api/evaluate-risk` accepts a JSON `POST` payload matching this validation schema:

| Field Name | Type | Description / Constraints |
| :--- | :--- | :--- |
| `order_id` | `int` or `str` | Unique tracking identifier for the transaction |
| `Shipping Mode` | `str` | Transit tier (e.g., `"Standard Class"`, `"First Class"`, `"Same Day"`) |
| `Order Region` | `str` | Geographic region (e.g., `"Southeast Asia"`, `"Western Europe"`, `"Central America"`) |
| `Customer Segment` | `str` | Segment identifier (e.g., `"Consumer"`, `"Corporate"`, `"Home Office"`) |
| `Days for shipment (scheduled)` | `int` or `float` | Allocated delivery window in days |
| `Product Price` | `int` or `float` | Unit cost of the item being shipped |
| `Order Item Quantity` | `int` | Quantity of items in the shipment |
| `Sales` | `int` or `float` | Total monetary value of the order (`Product Price * Order Item Quantity`) |

### 1.2 Output Schema (Server Response)
The API returns a JSON structure with HTTP status `200` upon successful inference and reasoning:

```json
{
  "status": "success",
  "ml_metrics": {
    "predicted_deviation_days": -0.14999526719867237,
    "late_risk_probability": 0.3501678268924717
  },
  "amber_advisory": "### **Logistics Risk Assessment Report...**"
}
```

## 2. Machine Learning Inference Pipeline

The ML pipeline is handled by three serialized binaries created during training and loaded globally on Flask startup:

```
[drishti_preprocessor.joblib] ➔ [drishti_regressor.joblib] ➔ (predicted_deviation_days)
                              ➔ [drishti_classifier.joblib] ➔ (late_risk_probability)
```

1.  **Preprocessor (`drishti_preprocessor.joblib`)**:
    *   Ingests a single-row Pandas DataFrame containing raw telemetry columns.
    *   Transforms raw numerical and categorical variables (e.g., encoding string values for `Shipping Mode`, `Customer Segment`, and `Order Region`).
2.  **Regressor (`drishti_regressor.joblib`)**:
    *   Accepts the processed numeric array.
    *   Outputs a scalar float representing the predicted delay deviation in days (positive numbers represent late shipments; negative numbers represent early/on-schedule arrivals).
3.  **Classifier (`drishti_classifier.joblib`)**:
    *   Accepts the processed numeric array.
    *   Evaluates the probability of fulfillment failure (SLA breach).
    *   Extracts index `1` of `predict_proba` to isolate the probability score.

## 3. Backend Architecture (`app.py`)

The backend is built as a Flask server running locally on `http://127.0.0.1:5000`.

### 3.1 Global Dependencies
*   **Pandas**: Reconstructs the raw JSON payload into structured DataFrames.
*   **Joblib**: Deserializes the machine learning pipelines into memory.
*   **Flask-CORS**: Configures cross-origin resource sharing to whitelist requests coming from the Next.js origin (`http://localhost:3000`).
*   **Google GenAI SDK**: Connects to the Gemini API.

### 3.2 Main Controller / API Gateway
*   **Route**: `/api/evaluate-risk` (`POST` method)
*   **Validation Track**: Checks for the existence of required properties and verifies data types inside `validate_payload(data)`. Returns HTTP `400` if validation fails.
*   **Inference Track**: Converts inputs, scales via `preprocessor`, runs regress/class models, and isolates float metrics.
*   **Reasoning Track (Amber Core)**:
    *   Persona: Senior enterprise supply chain logistics risk officer.
    *   Task: Convert statistical probability and deviation days into exactly two operational directives tailored to the order's regional node.
    *   Fallback mechanism: Directs requests to `gemini-3.5-flash` first; falls back to `gemini-3.1-flash-lite` if the primary model is unavailable.
    *   Graceful handling: If the GenAI gateway is offline, returns HTTP `500` containing the raw ML metrics along with a fallback message preserving structural integrity.

## 4. Frontend Architecture (Next.js Dashboard)

The user interface resides under `frontend/` and is developed as a static-ready Next.js application.

### 4.1 Dependency Stack
*   **Framer Motion**: Handles all micro-interactions, modal drawers, and status transitions.
*   **Tailwind CSS v4**: Utility styling, incorporating custom dark themes.
*   **Lucide React**: Vector icon system.
*   **React Markdown**: Renders the markdown output generated by Amber.

### 4.2 Core UI Components
1.  **TelemetryLogFeed**:
    *   Pill-shaped status widget in the header.
    *   Rotates through a static array of mock system events (e.g. S3 route cache, model weights checks) every 3.5 seconds.
    *   Implements fade exit/entry animations using `<AnimatePresence>`.
2.  **DecryptText (Matrix Decoder)**:
    *   Scrambles string values through alphanumeric symbols (`0X#A%...`) for a configurable window (`400ms`).
    *   Locks symbols from left to right as the animation completes to simulate real-time neural decryption.
3.  **Mitigation Action Dispatcher**:
    *   Renders buttons at the bottom of the reasoning console.
    *   Maps specific IDs (e.g., `12345`, `99821`) to distinct action items.
    *   Clicking a button overrides its layout to display a green glowing success indicator: `[ COMMAND ACCEPTED // ROUTING UPDATED ]`.
4.  **Raw Payload Inspector**:
    *   Sliding drawer panel that overlays the right-hand side of the viewport.
    *   Formats the combined request-response payload as JSON in a scrollable console.
    *   Features a one-click clipboard copying button.
